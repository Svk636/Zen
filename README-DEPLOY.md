# झेन जर्नल — Deployment Guide

## How It Works

```
Your computer          GitHub (public/private)    Netlify (your server)
─────────────          ───────────────────────    ─────────────────────
index.html    ──push──▶ index.html      ─────────▶ index.html
sw.js                   sw.js          auto-deploy sw.js
manifest.json           manifest.json              manifest.json
netlify.toml            netlify.toml               netlify.toml
config.js.example       config.js.example          config.js.example
schema.sql              schema.sql                 schema.sql
.gitignore              .gitignore
                                                   config.js  ← generated
config.js  (local only,                              at build time from
never committed)                                     env variables ✓
```

**GitHub holds your code. Netlify holds your secrets.**  
`config.js` is auto-generated by Netlify's build step — it never exists in git.

## Repository Structure

```
├── index.html            ← Main app (no secrets inside)
├── manifest.json         ← PWA manifest
├── sw.js                 ← Service Worker (offline + background sync)
├── schema.sql            ← Supabase DB setup — run once in SQL Editor
├── netlify.toml          ← Netlify build — generates config.js from env vars
├── config.js.example     ← Template for local dev reference (safe to commit)
├── .gitignore            ← Excludes config.js, .DS_Store, etc.

    ├── apple-touch-icon.png  (180×180)
    ├── favicon.ico
    └── screenshot-mobile.png  (390×844)
```

## One-Time Setup

### Step 1 — Push to GitHub

```bash
git init
git add .
# config.js is excluded by .gitignore — will NOT be pushed
git commit -m "initial commit"
```

Create a new repo at github.com (private or public — no secrets in the code), then:

```bash
git remote add origin https://github.com/yourname/zen-journal.git
git push -u origin main
```

Verify on GitHub that `config.js` is **not** there. Only `config.js.example` should appear.

---

### Step 2 — Connect Netlify to GitHub

1. Go to **netlify.com** → Log in → **Add new site** → **Import from Git**
2. Choose **GitHub** → Authorise → Select your repo
3. Build settings are read from `netlify.toml` automatically — leave them blank
4. Click **Deploy site**

The first deploy will produce an empty `config.js` (all keys blank) — that's fine, fix in Step 3.

---

### Step 3 — Add Your Secret Keys in Netlify

**Netlify Dashboard → Your site → Site configuration → Environment variables → Add variable**

Add these (only fill in the providers you actually use):

| Variable name | Value | Where to get it |
|---------------|-------|-----------------|
| `ZJ_PROVIDER` | `groq` | Which AI to use: `groq` / `gemini` / `openai` / `claude` |
| `ZJ_GROQ_KEY` | `gsk_xxxx...` | console.groq.com — free |
| `ZJ_GROQ_MODEL` | `qwen/qwen3-32b` | Recommended free model |
| `ZJ_GEMINI_KEY` | `AIzaS...` | aistudio.google.com — free |
| `ZJ_GEMINI_MODEL` | `gemini-1.5-flash` | |
| `ZJ_OPENAI_KEY` | `sk-proj-...` | platform.openai.com |
| `ZJ_OPENAI_MODEL` | `gpt-4o-mini` | |
| `ZJ_CLAUDE_KEY` | `sk-ant-...` | console.anthropic.com |
| `ZJ_CLAUDE_MODEL` | `claude-haiku-4-5-20251001` | |
| `ZJ_SUPABASE_URL` | `https://xxxx.supabase.co` | Supabase Dashboard → Project Settings → API |
| `ZJ_SUPABASE_ANON_KEY` | `eyJxxx...` | Supabase Dashboard → API → anon/public key |

> You only need **one** AI provider. Supabase is optional — app works fully offline without it.

After adding variables: **Deploys → Trigger deploy → Deploy site**

---

### Step 4 — Supabase DB Setup (optional, for cloud sync)

1. Create a free project at supabase.com
2. Go to **SQL Editor** → New query
3. Paste the contents of `schema.sql` and run it
4. Done — tables, indexes, RLS policies, and RPCs are all created

## Day-to-Day Workflow (After Setup)

```bash
# Make any change to index.html, sw.js, etc.
git add .
git commit -m "describe your change"
git push
# Netlify auto-detects the push and redeploys in ~30 seconds
# config.js is regenerated fresh from env variables on every deploy
```

You never touch Netlify manually again unless you need to change a secret key.

---

## Updating a Secret Key

1. Netlify Dashboard → Site configuration → Environment variables → Edit
2. Save → Deploys → Trigger deploy

---

## Local Development

```bash
cp config.js.example config.js
# Open config.js and fill in your actual keys for local testing
npx serve .
# or
python3 -m http.server 8080
```

`config.js` is gitignored — your local keys will never be accidentally committed.

---

## Security Notes

- `config.js` never exists in your git repo — excluded via `.gitignore`
- Netlify env variables are encrypted at rest, only exposed during the build
- The Supabase **anon key** is safe in the browser — RLS enforces per-user access
- The Supabase **service_role key** must NEVER go client-side — it bypasses RLS
- AI keys are browser-visible in the generated `config.js`. Use per-project keys with spending limits

## Cache Busting

When you make significant UI changes, bump the cache version in `sw.js` line 12:

```js
const CACHE_NAME = 'zj-v2';  // increment on major updates
```
