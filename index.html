०१ — झेन जर्नल

हे काय आहे आणि का वापरावे?

एक शांत, वेळ-निश्चित PWA जे तुमचे विचार, संकल्प, सवयी, DOJO प्रोटोकॉल्स आणि ज्ञान एकाच ऑफलाइन-फर्स्ट अ‍ॅपमध्ये जोडते.

✍️
जर्नल (१६ प्रकार)
विचार, संकल्प, कल्पना, प्रश्न, अंतर्दृष्टी, निर्णय, विश्वास, धोरण, कृती, प्रणाली, शिस्त, प्रतिबिंब, चूक, विजय, सवय — प्रत्येक प्रकाराचा स्वतंत्र रंग.
⏱
टाइमर सत्र
संकल्प निवडल्यावर AI मायक्रोस्टेप्ससह वेळ-निश्चित सत्र सुरू होते. दुहेरी रिंग टाइमर — मुख्य + प्रत्येक स्टेप.
⟳
सवय स्टॅक H1–H9
सकाळपासून रात्रीपर्यंत ९ हॅबिट ब्लॉक्स. AI वेळ वाटप करतो, दिवसाचा प्रकार ओळखतो, स्ट्रीक ट्रॅक करतो.
📋
DOJO OS टेम्पलेट्स
१२ जगण्याचे प्रोटोकॉल — सकाळ उघडणे ते मासिक फेज निकाल. AI सर्व फील्ड्स भरतो, विश्लेषण देतो.
🧠
द्वितीय मेंदू
[[लिंक्स]] आणि NLP वापरून 3D ज्ञान ग्राफ, Constellation दृश्य, Idea Engine, दैनंदिन Logseq-शैली नोंदी.
◎
AIE Analytics
Execution Score, Completion Rate, Consistency, Abandon Rate — CORRECTION / OPTIMIZATION / SCALING मोड. AI Directives.
☁️
Supabase सिंक
ऑफलाइन-फर्स्ट. Background Sync रांग. RLS सुरक्षा. सर्व उपकरणांवर एकसारखे.
📱
PWA (इंस्टॉल करा)
Chrome / Safari → "होम स्क्रीनवर जोडा" — अ‍ॅप सारखा अनुभव, कॅश केलेला, इंटरनेटशिवाय काम.
📱
PWA इंस्टॉल करा: Chrome वर address bar मध्ये ⊕ चिन्ह दिसेल — किंवा मेनू → "होम स्क्रीनवर जोडा". एकदा इंस्टॉल केल्यावर इंटरनेट नसताना सुद्धा संपूर्ण अ‍ॅप काम करतो.
०२ — सुरुवात

पहिल्यांदा कसे सुरू करावे?

अवघ्या ३ पावलांत सुरू करा — कोणतेही खाते, इंस्टॉलेशन किंवा API की आवश्यक नाही.

①
अ‍ॅप उघडा — लगेच काम करतो
Service Worker पहिल्या भेटीत स्वयंचलित स्थापित होतो. सर्व डेटा LocalStorage मध्ये जतन होतो — इंटरनेट नसताना सुद्धा. डेटा कुठेही जात नाही.
②
पहिली नोंद करा
मोठ्या textarea मध्ये टाइप करा. खालील प्रकार बटण निवडा (विचार, संकल्प, कल्पना…). — धरा — किंवा Ctrl+Enter दाबा.
③
AI + Cloud (पर्यायी, शिफारशीय)
तळाच्या सिंक टॅबमधून Groq की जोडा (मोफत: console.groq.com). Supabase खाते तयार केल्यास सर्व उपकरणांवर बॅकअप. config.js वापरल्यास keys deployment मध्ये सुरक्षित.

Ctrl + T

🎙
व्हॉइस इनपुट: मायक्रोफोन बटण (🎤) दाबा — बोला, transcript आपोआप textarea मध्ये येतो. मराठी / English दोन्ही भाषा उपलब्ध. Language selector खाली दिसतो.
०३ — जर्नल

विचार नोंदणी — १६ प्रकार

मुख्य स्क्रीन म्हणजे तुमचे स्विफ्ट capture interface. विचार लिहा, प्रकार निवडा, जतन करा — ३ सेकंदांत.

✍️
मोकळे लिहा
मनात जे येईल ते लिहा. [[संकल्पना]] वापरल्यास ते HawkEye ग्राफमध्ये नोड बनतात. #टॅग ग्राफ फिल्टरसाठी वापरता येतात.
🏷️
प्रकार — प्रत्येकाचा अर्थ
विचार — निरीक्षण, भावना. संकल्प → टाइमर सत्र उघडते. कल्पना — नवीन संकल्पना. प्रश्न — उत्तर हवे. अंतर्दृष्टी / निर्णय / विश्वास / धोरण / कृती / प्रणाली / शिस्त / प्रतिबिंब / चूक / विजय / सवय — प्रत्येकाचा रंग वेगळा, ग्राफमध्ये वेगळा नोड.
👆
नोंद संपादित करा
इतिहासातील कोणत्याही नोंदीवर टॅप करा → slide-up drawer उघडतो. मजकूर बदला, मायक्रोस्टेप्स पहा, किंवा नोंद हटवा. हटवणे soft-delete — Supabase मध्ये deleted_at सेट होतो, पुन्हा येत नाही.

०४ — संकल्प सत्र

संकल्प → कृती → पूर्ती

"संकल्प" प्रकार निवडल्यावर एक वेळ-निश्चित टाइमर सत्र उघडते — AI मायक्रोस्टेप्स सुचवतो, दुहेरी रिंग टाइमर चालतो.

📝
संकल्प लिहा — स्पष्ट, कृती-आधारित
उदा. "पुस्तकाचे ३ अध्याय वाचणे" · "प्रेझेंटेशनचे slides बनवणे" · "H2 सकाळ व्यायाम". संकल्प मजकूर AI ला दिला जातो.
⏱
वेळ सेट करा
तास : मिनिटे टाका. पोमोडोरो = ०:२५. खोल काम = १:३०. H1 ध्यान = १:०० (योग्य प्रकारे सेट केल्यास AIE completion rate सुधारतो).
＋
मायक्रोस्टेप्स जोडा
"+ पाऊल" बटण — प्रत्येक पाऊल drag-and-drop ने रचता येतो. प्रत्येक पावलाला स्वतंत्र वेळ दिल्यास inner ring त्या पावलाचा progress दाखवतो.
◈
AI पर्यावरण शिल्पकार
AI key असल्यास setup स्क्रीनवर आपोआप चालतो. तुमच्या संकल्पासाठी अपेक्षित अडथळे, पर्यावरण तयारी tasks आणि १२०-मिनिट execution plan दाखवतो.
＋
AI tasks → मायक्रोस्टेप्स
प्रत्येक AI task row वर "+ पाऊल" बटण दिसते — एक क्लिकने त्या task ला मायक्रोस्टेप म्हणून जोडता येते.
↻
पुन्हा सुचवा
"↻ पुन्हा" बटण — AI नवे suggestions generate करतो. H1–H9 संकल्पांसाठी habit stack context आपोआप जोडला जातो.
◯◯
दुहेरी रिंग टाइमर
बाहेरची रिंग = एकूण सत्र वेळ. आतली रिंग = सद्यस्थित पावलाचा वेळ. मध्यभागी HH:MM:SS countdown + सध्याचे step time.
⏸
थांबा / सोडा
"थांबा" = timer pause. "सोडा" = confirm dialog येतो — confirm केल्यास सत्र "abandoned" म्हणून जतन होते, AIE मध्ये abandon rate मध्ये मोजले जाते.
✓
पावले check करा
खालील steps list मध्ये प्रत्येक पाऊल tap करून ✓ done किंवा – skipped करा. AIE चा step completion rate हेच मोजतो.
🔔
घंटा — पूर्ती स्क्रीन
सत्र संपल्यावर ध्वनी संकेत. पूर्ती screen: done / skipped / pending counts. नोंद "completed" म्हणून जतन होते — AIE completion rate +१.
💭
"परत जा" → Journal
परत जाऊन लगेच reflection नोंदवा — हे HawkEye मध्ये त्याच दिवसाच्या नोडशी जोडले जाते. Patterns AI नंतर दाखवतो.

०७ — AI सहाय्यक

AI की जोडणे व कुठे वापरली जाते

चार providers उपलब्ध — Groq आणि Gemini मोफत. एकावेळी एकच provider सक्रिय. AI अ‍ॅपच्या प्रत्येक भागात आहे.

🔑
Groq की मिळवा (मोफत, शिफारशीय)
console.groq.com → Sign up → API Keys → Create. gsk_... ने सुरू होणारी key copy करा. दर महिना मोफत, खूप जलद.
⚙️
Runtime (UI मधून): सिंक टॅब → AI की
Provider निवडा → key paste करा → "जतन करा". LocalStorage मध्ये जतन होते. अ‍ॅप reload नाही लागत — लगेच वापर सुरू.
📄
Deployment (config.js): keys git मध्ये येत नाहीत
config.js file बनवा (खालील Deploy विभाग पहा). Priority: config.js > LocalStorage > blank. config.js नसल्यास app silently चालतो.
◈
AI पर्यावरण शिल्पकार (संकल्प setup)
अडथळे + prep tasks + 120-min execution plan. H1–H9 असल्यास habit stack context आपोआप prompt मध्ये जातो.
⏱
Habit AI Plan (सवय popup)
Target minutes मध्ये steps proportionally वाटप करतो. Context: आजचा दिवस, hour, recent journal. Insight: एक वाक्यात मार्गदर्शन.
📋
DOJO Templates — Joy Fill + विश्लेषण
सर्व fields एकाच वेळी भरतो, किंवा प्रत्येक field स्वतंत्र. Form विश्लेषण: तुमच्या उत्तरांवर sharp AI insight.
🧠

Idea Engine: तुमच्या नोंदींमधील hidden connections. AIE Directives: STOP / START / CONTINUE / PRIORITY_1 / PRIORITY_2 / INSIGHT — behavioral data वर based.

०९ — AIE Analytics

Adaptive Intelligence Engine — तुमची कार्यक्षमता मोजा

HawkEye → Analytics tab मध्ये तुमच्या journal behavior चे real-time analysis. कोणता mode आहात आणि काय करायला हवे ते AI सांगतो.

100
Execution Score (0–100)
Completion Rate · Step Rate · Consistency · Acting Ratio · Velocity · Abandon Rate — या ६ metrics चा weighted score. Score trend पाहा.
3 मोड
CORRECTION / OPTIMIZATION / SCALING
CORRECTION ⚠ = fundamentals broken, आधी basics. OPTIMIZATION ◎ = system चालतो, friction कमी करा. SCALING ▲ = सर्व metrics threshold वर — ambition वाढवा.
↑↓
Trend + Alerts
7-session moving average. CRITICAL alerts: completion rate <20%, consistency <20%, abandon >70%. WARNING: step rate <50%, velocity <0.3/day.
AI
AI Directives — STOP / START / CONTINUE
AI key असल्यास: STOP (हे करणे थांबवा) · START (हे सुरू करा) · CONTINUE (हे काम करतंय) · PRIORITY_1 · PRIORITY_2 · INSIGHT. Data-based, no filler.
📊
Score सुधारण्यासाठी: जेव्हा संकल्प "completed" म्हणून जतन होतो (टाइमर संपतो) तेव्हा completion rate वाढतो. "Abandon" कमी करा — छोटे संकल्प ठेवा जे पूर्ण होतात. Consistency = 14 दिवसांत active days.
१० — सिंक व डेटा

डेटा सुरक्षित — ऑफलाइन ते Cloud

Zen Journal offline-first आहे. LocalStorage = primary store. Supabase = optional cloud backup. Export = manual backup.

LocalStorage
आपोआप — इंटरनेट नसताना सुद्धा. शेवटच्या ९० नोंदी.

Background Sync
ऑफलाइन असताना नोंदी queue मध्ये. Internet आल्यावर SW आपोआप flush करतो.
JSON निर्यात
डेटा टॅब → सर्व नोंदी .json मध्ये. Ctrl+S shortcut.
CSV निर्यात
डेटा टॅब → Spreadsheet साठी .csv

☁️
Supabase सेटअप (५ मिनिटे)
१) supabase.com → मोफत खाते. २) SQL Editor → schema.sql paste करा → Run. ३) Project URL + anon key → index.html मध्ये paste. ४) अ‍ॅपमध्ये सिंक टॅब → Sign Up. ५) आपोआप sync सुरू.
🔒
RLS सुरक्षा — anon key safe आहे
Supabase Row-Level Security सक्रिय आहे — प्रत्येक policy auth.uid() वापरते. म्हणजेच anon key browser मध्ये असला तरी कोणी दुसरा तुमचा डेटा वाचू शकत नाही.
⚠️
Storage भरल्यास: "⚠ स्टोरेज भरले" flash दिसेल. डेटा टॅबमधून JSON export करा, नंतर जुन्या नोंदी delete करा. Supabase वापरल्यास हा issue येत नाही — unlimited cloud storage.
११ — Deploy & config.js

Production Deployment — Netlify / Vercel / GitHub Pages

Zen Journal एक single HTML file आहे — deploy करणे अत्यंत सोपे. Keys git मध्ये येऊ नयेत म्हणून config.js pattern वापरा.

📁
आवश्यक files
index.html · sw.js · manifest.json · icons/ (७ files). चारही root मध्ये एकत्र असाव्यात. schema.sql फक्त Supabase मध्ये run करायचे.
①
Netlify — zero config (शिफारशीय)
GitHub वर push → Netlify → Import from Git → Publish dir: . → Deploy. Environment variables मध्ये keys ठेवा, build-config.js वापरून config.js generate करा.
②
GitHub Pages — सर्वात सोपे
Settings → Pages → main branch. Keys index.html मध्ये hardcode करावे लागतात (किंवा GitHub Actions + build-config.js). HTTPS required for PWA.
📄
config.js — keys git बाहेर
index.html च्या शेजारी config.js बनवा. .gitignore मध्ये जोडा. नसल्यास app silently चालतो — keys UI मधून set करता येतात.
✦ config.js नमुना
window.APP_CONFIG = {
  provider: 'groq',
  groq: { key: 'gsk_xxxx…' },
  gemini: { key: 'AIza…' },
  openai: { key: 'sk-…', model: 'gpt-4.1-mini' },
  claude: { key: 'sk-ant-…', model: 'claude-haiku-4-5-20251001' },
  supabase: {
    url: 'https://xxxx.supabase.co',

🔒
Key priority order: config.js (highest) → LocalStorage (user-saved via UI) → hardcoded fallback (Groq only). Supabase service_role key कधीही client-side ठेवू नका — फक्त anon key safe.

छोटे संकल्प — पूर्ण होतात
AIE completion rate वाढवण्यासाठी: २ तासांचे एक मोठे संकल्प ऐवजी ३०-मिनिटांचे ४ संकल्प ठेवा. Abandon rate कमी होतो, score वाढतो.

🌙
H8 + H9 रात्र protocol
H8 reflection depth (word count, tags) हे next-day H1 quality चा सर्वात मोठा predictor आहे. H9 sleep onset = master cascade variable. या दोघांना protect करा.
🔄
